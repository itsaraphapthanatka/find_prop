เช็กลิสต์รีวิว/ทดสอบระบบ (QA Review)

จุดตรวจ (checkpoint) วางไว้ตอน "ปิดท้ายแต่ละ user journey ที่มีจุดตัดสินใจหรือผลลัพธ์ที่ตรวจสอบได้" — ไม่ใช่ทุกคลิก เพื่อให้ผู้ทดสอบไม่เหนื่อยและข้อมูลไม่รก

HOP · เอกสารทดสอบภายใน · ก่อนขึ้น production
วิธีใช้
  • ทำตามแต่ละ checkpoint แล้วกรอก 3 คอลัมน์ขวา: ผลจริง · สถานะ · หมายเหตุ/comment
  • เอาไป Google Sheet ได้: ลากเลือกทั้งตาราง → คัดลอก → วางใน Sheet (โครงตารางติดไปด้วย) · หรือกด Ctrl/Cmd+P พิมพ์เป็นกระดาษ (จัดเป็นแนวนอน A4 ให้แล้ว)
  • เจอสถานะ ❌ ไม่ผ่าน โดยเฉพาะที่ติดป้าย critical → ตั้ง ticket ส่งทีมพัฒนาทันที
สถานะ: ✅ ผ่าน❌ ไม่ผ่าน⚠️ มีข้อสังเกต critical = พังแล้วกระทบหนัก ยังไม่เคยทดสอบ

1 Flow เพิ่มทรัพย์

แยก checkpoint เพราะแต่ละจุดทดสอบคนละความสามารถ

Checkpoint (จุด)ผลที่คาดหวังผลจริงสถานะหมายเหตุ / comment
1.1 หลังกดปุ่ม "🎤 พูด" แล้วพูดข้อมูลทรัพย์ข้อความที่ถอดเสียงตรงกับที่พูด — ตัวเลข/หน่วยถูก (เช่น "แปดหมื่นห้า" → 85,000)
1.2 หลังกด "ให้ AI กรอกฟอร์ม" จากข้อความข้อมูลแมปเข้าช่องถูกทุกช่อง — โดยเฉพาะขนาดที่ดิน ≠ ขนาดอาคาร (ห้ามสลับ) จุดที่ AI เคยสับสน
จุดสำคัญมาก
1.3 หลังกด "บันทึก" ทรัพย์สำเร็จเปิดดูรายละเอียด/แก้ไข → ข้อมูลที่บันทึกจริงตรงกับที่กรอกครบทุกช่อง
1.4 วางหมุดบนแผนที่ → กด "+ เพิ่มทรัพย์ที่จุดนี้"ฟอร์มเปิดพร้อม lat/lng ตรงกับจุดที่ปักบนแผนที่

2 Flow แก้ไข & ลบทรัพย์

Checkpoint (จุด)ผลที่คาดหวังผลจริงสถานะหมายเหตุ / comment
2.1 หลังกด "บันทึก" การแก้ไขค่าที่แก้เปลี่ยนตามจริง · ค่าอื่นที่ไม่ได้แตะไม่เพี้ยน (เทียบก่อน–หลัง)
2.2 ก่อนกดปุ่ม "ลบ" — หยุดรีวิวก่อนคลิก criticalกดลบต้องมี dialog ยืนยันก่อนเสมอ ทุกที่ (หน้ารายการ + แผงรายละเอียด + ในแอปมือถือ) ถ้าลบทันทีไม่ถาม → flag critical
โค้ดปัจจุบันมี window.confirm "ลบรายการ …?" แล้ว — จุดนี้ยืนยันว่าเด้งจริงทุกหน้า/ทุกอุปกรณ์

3 Flow ค้นหา & กรอง

Checkpoint (จุด)ผลที่คาดหวังผลจริงสถานะหมายเหตุ / comment
3.1 ช่องค้นหาบน headerพิมพ์รหัส/ทำเล/ประเภท → กรองรายการถูกต้องและทันที
3.2 เทียบช่องค้นหาหลัก vs dropdown ค้นหาใน เปรียบเทียบ / แผนเยี่ยม / AIพิมพ์คำเดียวกันทั้งสองที่ → ได้ผลลัพธ์สอดคล้องกัน (ชุดทรัพย์เดียวกัน)
เคยเจอความไม่สอดคล้องระหว่างช่องหลักกับ dropdown

4 Flow ที่ใช้ AI ช่วยวิเคราะห์

ทุกจุด: หลังกดให้ AI ประมวลผล เช็ก 2 เรื่อง — (ก) คำตอบสมเหตุสมผลกับข้อมูลจริง ไม่มโนทรัพย์ที่ไม่มี · (ข) เวลาตอบสนองยอมรับได้ (ระบุวินาที)

Checkpoint (จุด)ผลที่คาดหวังผลจริงสถานะหมายเหตุ / comment (เวลา…วิ)
4.1 เปรียบเทียบทรัพย์ — AI เขียนบทวิเคราะห์บทวิเคราะห์อิงสเปกจริงของทรัพย์ที่เลือก · จุดเด่น/ข้อควรพิจารณาสมเหตุสมผล
4.2 Dashboard — AI วิเคราะห์พอร์ตรายวันสรุปตรงกับข้อมูลพอร์ตจริง (จำนวน/สัดส่วน/ช่องว่างความต้องการ)
4.3 แผนเยี่ยม — ลูกค้าเปลี่ยน requirement → AI re-matchแนะนำทรัพย์ที่ตรง requirement ใหม่จริง + ชี้จุดเดิมที่ไม่ตรง (รหัสมีจริงในระบบ)
4.4 AI Assistant — แชทถาม/สั่งงานตอบจากข้อมูลจริง (ไม่มีก็บอกตรง) · คำสั่งมีปุ่มยืนยันก่อนทำจริง

5 Flow ความเสี่ยงสูง (ทำผิดกระทบมาก)

ต้องยืนยันก่อนลงมือจริง + ตรวจว่ามี validation กันข้อมูลผิด

Checkpoint (จุด)ผลที่คาดหวังผลจริงสถานะหมายเหตุ / comment
5.1 เพิ่มลูกทีม — ก่อนกด "เพิ่มลูกทีม"ใช้อีเมล/ข้อมูล ทดสอบ เท่านั้น (ไม่สร้างบัญชีจริงมั่ว) · ทวนก่อนกด
5.2 เพิ่มลูกทีม — validation ยังไม่เคยทดสอบระบบกันอีเมลซ้ำ/รหัสสั้นเกิน 6/ช่องว่าง — ขึ้น error ชัด ไม่สร้างบัญชีพัง
5.3 นำเข้า (Import) — ก่อนกด upload จริงใช้ไฟล์ ทดสอบ เท่านั้น · ทวนก่อนนำเข้า
5.4 นำเข้า — validation ยังไม่เคยทดสอบกันข้อมูลผิด: คอลัมน์ไม่ครบ/ชนิดผิด/รหัสซ้ำ (ให้เลือกข้าม/อัปเดตทับ) — ไม่นำเข้าขยะเข้าฐาน
HOP — เช็กลิสต์รีวิว/ทดสอบระบบ · ยึดหลัก "ปิดท้าย journey ที่มีจุดตัดสินใจ/ผลตรวจสอบได้" · เฟสถัดไป: ฝัง review เข้าแอป (super เปิด-ปิดได้)
